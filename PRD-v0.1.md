## 1. 产品愿景

**打造世界上最优秀的文本翻译网站，让 AI 翻译超越并取代专业的人工翻译。**

### 1.1 核心价值主张
| 维度 | 专业人工翻译 | GoodTrans AI | 优势 |
|------|-------------|--------------|------|
| **准确性** | 95%+ | 98%+（5 轮优化） | +3% |
| **速度** | 2000-3000 字/小时 | 10 万 + 字/分钟 | 2000 倍 |
| **成本** | ￥0.10-0.30/字 | ￥0.0075/字（75% 折扣） | 10-30 倍 |
| **一致性** | 需人工校对 | TM+ 术语库强制 | 100% |
| **可用性** | 需预约排期 | 7×24 即时 | 随时可用 |

## 2. 用户旅程地图

### 2.1 完整用户旅程

```
┌─────────────────────────────────────────────────────────────────┐
│  发现 → 了解 → 决策 → 下单 → 等待 → 交付 → 留存/裂变            │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 各阶段详细设计

| 阶段 | 用户在哪 | 用户看到什么 | 用户感受/疑虑 | 我们的应对 |
|------|----------|--------------|--------------|-----------|
| **发现** | 知乎/小红书搜索"翻译" | AI 翻译 vs 人工翻译对比文章 | "AI 翻译能有多准确？" | 深度测评文章 + 数据对比 |
| **了解** | 进入 GoodTrans 官网 | 5 轮翻译引擎说明 + 案例展示 | "真的比人工好吗？有证据吗？" | 真实案例 + 用户评价 + 免费试用 |
| **决策** | 查看定价 + 免费试用 | 免费试翻 500 字 | "先试试水，不好用就跑" | 无门槛试用 + 即时出结果 |
| **下单** | 上传文档 + 选择语言 | 简单表单 + 即时报价 | "会不会很复杂？" | 3 步完成（上传→选择→支付） |
| **等待** | 等待翻译完成 | 进度通知 + 预计完成时间 | "什么时候能好？" | 实时进度 + 邮件/微信通知 |
| **交付** | 收到翻译文档 | 原文 + 译文对照 + 质量报告 | "质量到底怎么样？" | 质量评分 + 术语一致性报告 |
| **留存** | 考虑是否复购 | 优惠券 + 新语言推荐 | "下次还来吗？" | 复购优惠 + 会员体系 |

## 4. 技术架构

### 4.1 整体架构图

```
┌─────────────────────────────────────────────────────────────────┐
│                        前端层                                    │
│  Next.js + Tailwind CSS（SEO 友好，开发速度快）                   │
├─────────────────────────────────────────────────────────────────┤
│                        后端层                                    │
│  Python FastAPI / Node.js                                       │
│  用户认证：Clerk / Firebase Auth                                │
│  数据库：Supabase（PostgreSQL）                                  │
│  文件存储：阿里云 OSS                                            │
│  支付：微信/支付宝 H5 支付                                       │
├─────────────────────────────────────────────────────────────────┤
│                       AI 翻译层                                  │
│  5 轮翻译引擎：OpenRouter / 302.ai 多模型路由                     │
│  任务队列：Celery / BullMQ（异步处理）                           │
│  翻译记忆库：LanceDB / PostgreSQL                               │
│  术语库：PostgreSQL + 模糊匹配                                   │
├─────────────────────────────────────────────────────────────────┤
│                      基础设施                                    │
│  部署：Vercel（前端）+ Railway/Render（后端）                   │
│  监控：Sentry（错误告警）+ Uptime Robot（可用性）               │
│  日志：Better Stack / Logtail                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 关键技术决策

| 功能 | 选项 A | 选项 B | 决策 | 理由 |
|------|--------|--------|------|------|
| 用户认证 | Clerk | 自建 | ✅ Clerk | 10 分钟集成，安全省心 |
| 数据库 | Supabase | 自建 PostgreSQL | ✅ Supabase | 免费额度够用，可视化界面 |
| 文件存储 | 阿里云 OSS | 自建 | ✅ 阿里云 OSS | 便宜稳定，国内速度快 |
| 邮件推送 | Resend | 自建 SMTP | ✅ Resend | 简单好用，送达率高 |
| 监控告警 | Sentry | 自建 | ✅ Sentry | 出错自动通知，生态成熟 |
| 翻译引擎 | OpenRouter | 302.ai | ✅ 两者都用 | 多模型路由，成本优化 |

### 4.3 自动化链路设计

```
用户上传文档
    ↓
OSS 存储 → 获得文档 URL
    ↓
创建订单记录（待支付）
    ↓
用户支付 → 支付回调
    ↓
订单状态更新为"已支付"
    ↓
触发翻译任务队列
    ↓
┌─────────────────────────────────────┐
│         5 轮翻译引擎                  │
│  1. 格式优化 → 2. 初稿翻译           │
│  3. 反思评审 → 4. 终稿优化           │
│  5. 质量验证                         │
└─────────────────────────────────────┘
    ↓
翻译结果上传 OSS
    ↓
更新订单状态为"完成"
    ↓
邮件/微信通知用户
    ↓
用户下载翻译结果
```

## 5. 定价策略

### 5.1 成本结构分析

假设翻译 1000 字：
- **5 轮翻译** ≈ 5x token 消耗
- **OpenRouter API** ≈ $0.002/1000 tokens
- **1000 字** ≈ 500-800 tokens × 5 轮 ≈ $0.005-0.01
- **直接成本** ≈ ￥0.05-0.1 元/千字

### 5.2 人工翻译定价参考

| 类型 | 价格（元/千字） |
|------|----------------|
| 普通文档 | ￥100-150 |
| 专业文档 | ￥200-300 |
| 文学翻译 | ￥300-500 |
| 同传翻译 | ￥1000+ |

**取中间值￥200/千字作为基准**

### 5.3 GoodTrans 定价

| 套餐 | 价格 | 字数额度 | 单价（元/千字） | 目标用户 |
|------|------|----------|----------------|----------|
| **Free** | ￥0/月 | 1 千字 | 免费 | 体验用户 |
| **个人** | ￥99/月 | 10 万字 | ￥0.99 | 自由职业者 |
| **专业** | ￥499/月 | 100 万字 | ￥0.50 | 中小企业 |
| **企业** | 定制 | 定制 | 面议 | 大企业 |

**定价逻辑**：
- 个人套餐：￥99/10 万字 = ￥0.99/千字 ≈ 人工的 **0.5%**
- 专业套餐：￥499/100 万字 = ￥0.50/千字 ≈ 人工的 **0.25%**
- 远低于人工翻译的 75%，但毛利仍高达 **85-95%**

### 5.4 价格锚定策略

在定价页面展示三个套餐：
```
┌─────────────┬─────────────┬─────────────┐
│   个人版    │  专业版 ⭐   │   企业版    │
│   ￥99/月   │  ￥499/月   │   定制      │
│  10 万字    │  100 万字   │   定制      │
│  ￥0.99/千  │  ￥0.50/千  │   面议      │
└─────────────┴─────────────┴─────────────┘
```

**专业版**作为最想卖的产品，用个人版和企业版作为锚点。

## 6. 质量验收标准

### 6.1 翻译质量指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| **准确性** | 98%+ | 人工抽检 100 篇 |
| **术语一致性** | 100% | 术语表强制匹配 |
| **格式完整性** | 99%+ | 自动对比原文格式 |
| **漏译率** | <0.1% | 段落数量对比 |
| **用户满意度** | 4.5/5+ | 用户评价 |

### 6.2 5 轮翻译引擎质量标准

| 轮次 | 输入 | 输出 | 质量检查点 |
|------|------|------|-----------|
| **1. 格式优化** | 原始文档 | Markdown 格式 | 保留原文结构 |
| **2. 初稿翻译** | Markdown | 译文初稿 | 信达雅标准 |
| **3. 反思评审** | 原文 + 初稿 | 改进清单 | 找出 6+ 个问题 |
| **4. 终稿优化** | 初稿 + 清单 | 优化译文 | 逐条应用改进 |
| **5. 质量验证** | 终稿 | 质量报告 | 术语/漏译/格式检查 |

## 8. 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| **翻译质量不稳定** | 中 | 高 | 5 轮优化 + 质量验证 + 人工抽检 |
| **API 成本超预算** | 中 | 中 | 多模型路由 + TM 复用 + 限流 |
| **冷启动获客困难** | 高 | 高 | CEO 亲自跟进 + 手动运营前 100 用户 |
| **支付接入复杂** | 中 | 中 | 用现成 SaaS 工具（如 Stripe） |
| **并发性能不足** | 低 | 中 | 任务队列 + 异步处理 + 弹性扩容 |
