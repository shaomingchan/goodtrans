## 二、Animaker.AI 登录问题修复

### 2.1 问题诊断

**CEO 发现**: "我已经找到问题了"

**根本原因**: www 与非 www 域名不一致导致 Cookie 丢失

**问题详情**:
- 用户在 `www.animaker.dev` 发起 Google 登录
- Google 回调到 `animaker.dev`（没有 www）
- NextAuth 的 PKCE 验证码存在 `www.animaker.dev` 的 Cookie
- 跨子域读取不到 Cookie，验证失败

---

### 2.2 解决方案

**CEO 选择**: 方案一（强制统一域名）

**实施内容**:
- 在 middleware.ts 添加 www → 非 www 的 301 重定向
- 确保登录发起和回调落在同一个域下

**代码修改**:
```typescript
// middleware.ts
const host = req.headers.get('host')
if (host?.startsWith('www.')) {
  const url = req.nextUrl.clone()
  url.host = host.replace('www.', '')
  return NextResponse.redirect(url, 301)
}
```
