## 第一轮：各部门问题清单

### 🔧 DevOps（运维与财务）

#### P0 阻断问题
1. **冷启动延迟** - Vercel Serverless 翻译任务超时，需异步化
2. **无健康检查** - API 挂了不知道，用户无声无息失败
3. **环境变量混乱** - Stripe 测试/生产密钥混用风险
4. **Stripe Webhook 未配置** - 支付成功但积分不发放（最常见问题）
5. **Stripe 测试/生产模式未隔离** - 资金安全风险

#### P1 问题
- API 缓存去重（节省 20-40% 费用）
- Presigned URL 直传（节省 80%+ 带宽）
- 防刷速率限制
- Stripe Tax 开启（税务合规）

#### P2 问题
- R2 生命周期策略
- 退款/争议处理

**总工时**: 8-12h (P0) + 18-24h (P1)

---

### 💻 Dev（开发）

#### P0 阻断问题
1. **翻译 API 无认证保护** - 裸奔的 Claude API Key，任何人都能刷爆
2. **支付 checkout 绕过 ShipAny 体系** - 不创建 Order、不发放 Credits、Webhook 断开
3. **SSRF 漏洞** - `/api/proxy/file` 可访问内网服务
4. **API Key 泄露** - `.env.development` 包含真实凭据

#### P1 问题
- 两套定价页面冲突
- 翻译不扣 Credits
- 翻译页面风格断裂
- **PDF 不支持中文** - jsPDF 只支持 Latin 字符，翻译产品致命问题
- 翻译 API 无流式输出（30-60 秒干等）
- max_tokens 硬编码 4096（长文档被截断）
- 0 个测试文件

#### P2 问题
- 没有 middleware 路由保护
- 没有安全 Headers
- 大量 ShipAny 冗余代码
- 错误处理泄露敏感信息
- 计费单位混乱（word vs char）

**核心诊断**: "ShipAny 模板 + 一个裸的翻译页面，两套系统还没接通"

**总工时**: 8-12h (P0) + 18-24h (P1) + 6-8h (P2)

### 📊 Research（调研）

#### 综合评级: C+
**核心问题**: GoodTrans 定位模糊，在竞争激烈的赛道中缺乏明确护城河

#### 竞品威胁
| 竞品 | 定价 | 核心优势 | 弱点 |
|------|------|----------|------|
| DeepL Pro | $8.74/月起 | 翻译质量顶级，格式保留好 | 中文质量差，价格高 |
| Google Translate | 免费 | 覆盖 135 语言，零门槛 | 质量一般，格式经常乱 |
| ChatGPT/Claude | $20/月 | 用户习惯已养成，质量好 | 无格式，无批量 |

#### 核心痛点（按严重程度）
1. **格式崩了**（⭐⭐⭐⭐⭐）- PDF/Word 翻译后排版乱
2. **专业术语翻错**（⭐⭐⭐⭐⭐）- 法律、医学、技术文件
3. **批量翻译效率低**（⭐⭐⭐⭐）
4. **数据安全/隐私顾虑**（⭐⭐⭐⭐）

#### 定价策略建议
```
Freemium 入口：
  - 免费：每月 5 页/5 个文件

Pro 个人：
  - $9/月 → 100 页/月
  - 或 $2.99/文件（按需，低频用户买单）

Team：
  - $29/月 → 500 页 + 术语表 + 协作审阅
```

#### P0 行动建议
1. **先跑通一个真实付费用户**（哪怕手动处理文件）
2. **Stripe Payment Link 快速上线**（5 分钟接通收款）
3. **Landing Page 聚焦格式保留**（加翻译前后对比图）
4. **找 5 个真实用户验证痛点**

#### 差异化方向
- **垂直赛道**：法律文件翻译 / 技术手册本地化 / 中文↔小语种
- **格式保留**：这是可量化的差异点（用对比图证明）
- **按需付费**：低频用户友好（$3-5/文件）
- **翻译记忆库**：用户翻译过的内容积累成资产（留存飞轮）

### 📝 Content（内容运营）

#### P0 问题
1. **价值主张不具体** - "文档翻译工具"是品类不是理由
2. **定价页缺锚点** - 没有对比参照物（人工翻译 vs AI）
3. **目标用户不明确** - 文案在对所有人说话，没人觉得"这是为我做的"
4. **Onboarding 引导缺失** - 用户上传后不知道下一步
5. **隐私/安全背书缺失** - 企业用户最大顾虑
6. **核心 FAQ 缺失** - 用户遇到问题找不到答案

#### P1 问题
- 主关键词竞争过高（需要深挖长尾意图词）
- 语言对目标词未布局（大量高意图搜索流量流失）
- 文件格式目标词未覆盖
- Blog/内容资产缺失（无法积累 SEO 权重）

#### 核心建议
**选定 1-2 个最小可行受众**（Smallest Viable Audience）：
- 跨境电商卖家（产品说明书/合规文件）
- 法务/合规团队（合同/协议翻译）

**一句话总结**: GoodTrans 现在最大的内容问题不是"写得不好"，是"在对所有人说话，结果没人听"。
