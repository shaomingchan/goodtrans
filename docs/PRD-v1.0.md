## 一、产品概述

### 1.1 产品定位

GoodTrans 是一款基于 AI 反思翻译技术的 SaaS 产品，致力于提供超越人工翻译质量的专业翻译服务。

**核心使命**：
- 中文：愿世上没有文字的障碍
- 英文：No language should be a barrier

**产品愿景**：
打造世界上最优质的翻译工具，质量超越人工翻译。

### 1.2 核心价值主张

**相比传统机器翻译（DeepL/Google Translate）**：
- ✅ 5 轮反思翻译，质量更高
- ✅ 三层术语库系统，术语 100% 一致
- ✅ 支持 100+ 语言
- ✅ 按需付费，无需订阅

**相比人工翻译**：
- ✅ 速度快 100 倍（10 万字 12 分钟 vs 数天）
- ✅ 成本低 3-8 倍（$0.025/字 vs $0.1-0.3/字）
- ✅ 术语 100% 一致（人工翻译约 70%）
- ✅ 质量稳定（AI 不会疲劳）
- ✅ 无限规模化

### 3.2 核心功能详解

#### 功能 1：5 轮反思翻译引擎

**第 1 轮：PDF 提取**
- 工具：MinerU（开源 PDF 解析）
- 输入：PDF/Word/图片
- 输出：Markdown 格式文本
- 时间：5-30 秒

**第 2 轮：格式优化**
- 模型：Claude Sonnet 4.6
- 任务：清理冗余、标准化 Markdown
- 时间：5-10 秒

**第 3 轮：翻译初稿**
- 模型：Claude Sonnet 4.6
- 任务：专业翻译（信达雅标准）
- 支持：强制使用术语表
- 时间：10-20 秒/段

**第 4 轮：翻译反思**
- 模型：Claude Opus 4.6
- 任务：评估质量，找出问题
- 输出：改进建议清单
- 时间：10-20 秒/段

**第 5 轮：翻译终稿**
- 模型：Claude Opus 4.6
- 任务：应用改进，输出最终译文
- 输出：高质量译文（Markdown）
- 时间：10-20 秒/段

**总时间**：10 万字约 12 分钟

#### 功能 2：三层术语库系统

**第一层：用户私有术语库（最高优先级）**
- 用户自定义术语
- 100% 强制使用
- 支持导入/导出

**第二层：官方术语库（中等优先级）**
- 按行业分类（技术/医学/法律/金融/工程）
- 预置 10 万条术语
- 持续更新

**第三层：众包术语库（最低优先级）**
- 用户贡献的术语
- 社区投票验证
- 未来功能

**术语提取流程**：
1. 反向匹配：在文档中搜索术语库中的术语（1 秒）
2. AI 补充：Opus 识别新术语（30 秒）
3. 用户确认：展示术语表，用户可修改
4. 翻译集成：每段翻译都强制使用术语表

#### 功能 3：翻译记忆（TM）

**核心机制**：
- 存储历史翻译片段
- 相似内容自动复用
- 降低成本，提高一致性

**实现方式**：
- 向量数据库存储（LanceDB）
- 语义相似度检索
- 匹配度 > 90% 直接复用

**未来功能**（Phase 2）

#### 功能 4：质量评估

**自动检查项**：
1. 术语一致性（100%）
2. 格式完整性（段落、标题、列表）
3. 数字准确性（日期、金额、百分比）
4. 专有名词保留（人名、地名、公司名）

**质量报告**：
- 术语使用统计
- 质量评分（0-100）
- 潜在问题标记

#### 功能 5：多语言支持

**支持语言**：100+ 语言（Claude Opus 全语言支持）

**重点语言**：
- 中文 ↔ 英文
- 中文 ↔ 日文
- 中文 ↔ 韩文
- 英文 ↔ 西班牙文
- 英文 ↔ 法文

**IP 自动检测**：
- 使用 Vercel Geo API
- 日本 IP → 日文版
- 中国 IP → 中文版
- 欧美 IP → 英文版

### 5.2 数据库设计

**核心表结构**：

**users 表**：
```sql
- id (uuid, primary key)
- email (string, unique)
- name (string)
- credits (integer) -- 剩余字数
- created_at (timestamp)
- updated_at (timestamp)
```

**translations 表**：
```sql
- id (uuid, primary key)
- user_id (uuid, foreign key)
- source_language (string)
- target_language (string)
- industry (string) -- tech/medical/legal/finance/engineering
- word_count (integer)
- status (enum) -- pending/processing/completed/failed
- glossary (jsonb) -- 术语表
- source_file_url (string)
- result_file_url (string)
- quality_score (integer) -- 0-100
- created_at (timestamp)
- completed_at (timestamp)
```

**glossaries 表**：
```sql
- id (uuid, primary key)
- user_id (uuid, foreign key, nullable) -- null = 官方术语库
- industry (string)
- term (string)
- translation (string)
- source_language (string)
- target_language (string)
- usage_count (integer)
- created_at (timestamp)
```

**payments 表**：
```sql
- id (uuid, primary key)
- user_id (uuid, foreign key)
- amount (decimal)
- credits (integer) -- 购买的字数
- payment_method (string) -- creem/stripe
- status (enum) -- pending/completed/failed
- created_at (timestamp)
```

## 六、商业模式

### 6.1 定价策略

**字数包定价**：
- **1 万字包**：$250（$0.025/字）
- **5 万字包**：$1,200（$0.024/字，4% 折扣）

**免费额度**：
- 普通注册：500 字（体验）
- 推荐注册：2,000 字（体现专业度）

**推荐奖励**：
- 推荐用户充值，双方各得 10% 额度

### 6.2 定价逻辑

**对标人工翻译**：
- 人工翻译：$0.1-0.3/字（$10,000-30,000/10 万字）
- GoodTrans：$0.025/字（$2,500/10 万字）
- **价格为人工翻译的 25%，质量超越人工**

**对标机器翻译**：
- DeepL Pro：$30/月订阅（无限量，但质量一般）
- GoodTrans：按需付费，质量更高

### 6.3 收入预测

**假设**：
- 首批 100 用户（8 周获取）
- 平均每用户每月翻译 5 万字
- 平均客单价：$1,200/月

**收入预测**：
- 第 1 个月：10 用户 × $1,200 = $12,000
- 第 2 个月：30 用户 × $1,200 = $36,000
- 第 3 个月：100 用户 × $1,200 = $120,000

**成本预测**：
- AI 成本：$120,000 × 13.4% = $16,080
- 基础设施：$500/月
- **毛利润**：$120,000 - $16,080 - $500 = $103,420

## 七、竞争分析

### 7.1 主要竞争对手

**DeepL**：
- 优势：翻译质量较好，支持 31 种语言
- 劣势：无术语库，无反思翻译，订阅制
- 定价：$30/月（Pro 版）

**Google Translate**：
- 优势：免费，支持 100+ 语言
- 劣势：质量一般，无专业功能
- 定价：免费

**人工翻译平台（Gengo/Unbabel）**：
- 优势：人工翻译，质量有保障
- 劣势：速度慢，成本高
- 定价：$0.1-0.3/字

### 7.2 竞争优势

| 维度 | DeepL | Google | 人工翻译 | GoodTrans |
|------|-------|--------|---------|-----------|
| 翻译质量 | 7/10 | 6/10 | 8/10 | 9/10 ✅ |
| 术语一致性 | 70% | 60% | 70% | 100% ✅ |
| 处理速度 | 快 | 快 | 慢 | 快 ✅ |
| 成本 | 订阅制 | 免费 | 高 | 按需付费 ✅ |
| 多语言 | 31 种 | 100+ | 有限 | 100+ ✅ |
| 用户术语库 | ❌ | ❌ | ✅ | ✅ |
| 反思翻译 | ❌ | ❌ | ❌ | ✅ |

### 7.3 差异化策略

1. **质量优先**：5 轮反思翻译，质量超越人工
2. **术语 100% 一致**：三层术语库系统
3. **按需付费**：用多少付多少，无需订阅
4. **开发者友好**：API-first 设计（Phase 3）
5. **中文场景优化**：深度理解中文语境

## 十、风险与应对

### 10.1 技术风险

**风险 1：AI 成本过高**
- 应对：优化 Prompt，减少 token 消耗
- 应对：引入缓存机制，复用历史翻译

**风险 2：翻译质量不稳定**
- 应对：建立质量监控系统
- 应对：用户反馈机制，持续优化

**风险 3：处理速度慢**
- 应对：并行处理，多段同时翻译
- 应对：优化分段策略

### 10.2 商业风险

**风险 1：用户获取成本高**
- 应对：精准定位目标用户
- 应对：内容营销 + 社区运营
